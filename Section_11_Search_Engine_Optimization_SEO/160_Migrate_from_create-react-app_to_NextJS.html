📚Material-UI Next.js Boilerplate

Go to Material-UI docs → Guides → Server Side Rendering, scroll to the Next.js example, and run the provided curl command to download the starter. 📥

⚙️Install & Run

Inside the new next.js folder, run npm install to fetch deps, then npm run dev to start the server at localhost:3000. 🏃‍♂️

🗂️Pages → File System Routing

In /pages, index.js maps to “/” and about.js maps to “/about”. Add doctor.js and visit “/doctor” automatically—no extra router setup! 🚪

🔗Next.js Link Wrapper

Use the provided src/Link.js component instead of React Router’s link. Pass href="..." just like a regular anchor tag. 🔌

📁Copy UI Folder

Copy your old UI folder into src/UI. Remove the boilerplate theme.js and update imports in pages/_app.js to point at UI/theme.js. ✂️

⚙️_app.js & _document.js

The special pages/_app.js and pages/_document.js wrap all pages—like CRA’s App.js. Use them for global themes & SSR setup. 🛠️

🌐Public Folder for Static Assets

Create a public/ folder at project root. Place images under public/assets/ and reference via /assets/filename.svg in <img src="/assets/..." />. No JS import needed! 📂

✏️Rename Assets

Remove spaces from filenames (e.g., custom software icon.svg → custom-software.svg) to avoid URL issues. ⚠️

📂Copy Assets Into Public

Copy your entire assets/ folder into public/. Now http://localhost:3000/assets/your-file.svg serves directly. 🚚
