🚀Next.js Boilerplate Setup

1. In Material-UI docs → Guides → Server Side Rendering, scroll to “Next.js Example.”
2. Copy the shown curl command and run it to download the starter code. 📥

⚙️Install & Run Dev Server

Inside the new next.js folder:
• npm install to fetch dependencies
• npm run dev to start at localhost:3000 🖥️

🗂️File-System Routing

Put pages in /pages:
• pages/index.js → “/”
• pages/about.js → “/about”
Add doctor.js → “/doctor” automatically—no extra router! 📁

🔗Link Component

Use the boilerplate’s src/Link.js instead of React Router. Pass href="/path" like a normal link. 🔄

📂Copy UI Folder

Copy your old UI/ folder into src/UI/. Remove the boilerplate theme and update imports in pages/_app.js to UI/theme.js. ✂️

🌐Static Assets in public/

Create a public/ folder. Inside it, put an assets/ subfolder. Reference images as:

<img src="/assets/my-image.svg" />—served directly, no JS import. 📂

✏️Rename Filenames

Remove spaces: e.g. custom software icon.svg → custom-software.svg. Avoid URL issues. 🚧

💾Copy Assets

Copy your old assets/ folder into public/assets/. Now /assets/… paths work in both browser & server. 📤

🎞️Animations Folder

Copy your animations/ (JS files) into src/animations/, not public/—so Babel/Webpack still process them. 📦

🛠️Update Component Imports

In pages, replace any UI imports from "../UI/..." paths, updating to "../src/UI/...". Also change image imports to static URLs under /assets/…. 🔧
