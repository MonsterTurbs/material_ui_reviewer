🔍 Overview

Learn how to turn your styled-components syntax into Material UI’s built-in styled() function, so you can fully embrace MUI’s component ecosystem. 🚀

1️⃣ Identify Original Styled-Component

Styled-Components uses:

import styled from 'styled-components'; const LoginInput = styled.input` padding: 50px 20px; margin-bottom: 50px; width: 400px; text-align: center; border-radius: 4px; `;

This wraps a vanilla HTML <input> with custom CSS. 💅

2️⃣ Switch to MUI’s styled()

Import MUI’s styled from the theme/styles package:

import { styled } from '@mui/material/styles';

3️⃣ Convert Syntax

Wrap element/tag in styled(), then provide an object of styles:

const LoginInput = styled('input')({ padding: '50px 20px', marginBottom: '50px', width: 400, textAlign: 'center', borderRadius: 4, });
  • HTML tag → string, e.g. 'input'
  • Component → direct reference, e.g. Button
  • Style strings → object keys → camelCase
  • Multiple CSS values → quoted string '50px 20px'
  • Single numeric value → number (MUI adds px)

4️⃣ Think in Components

Whenever possible, replace raw HTML tags (<div>, <input>) with MUI components (Grid, TextField, Button), and style those instead:

const LoginContainer = styled(Grid)({ display: 'flex', alignItems: 'center', justifyContent: 'space-between', });

This ensures you leverage MUI’s built-in layout, theming, and responsiveness. 📐

5️⃣ Preview & Tweak

After conversion, test and adjust any mismatched styles. MUI components may have default paddings, so review on multiple screen sizes. 🔎

🎉 Result

You now have a pure Material UI styling approach—no extra dependencies, full theming, and cohesive layouts—ready to build faster! 🌟
