🎯 Goal

Make the estimate page jump to each question’s title on mobile—so users never lose context! 📱✨

📱 Why Snap On Mobile?

On small screens, navigating questions can hide the title. Snapping brings the title back into view automatically. 👀🔝

🛠️ Import useRef Hook

import React, { useState, useRef, ... } from 'react';

useRef gives a handle to the actual DOM node so we can measure and scroll to it.

🔗 Create & Attach the Ref

Generate the ref and attach to the wrapper of your question title:

const questionRef = useRef(null); // in JSX: <Grid item ref={questionRef}> <Typography>{currentQuestion.title}</Typography> </Grid>

🔄 Scroll on Navigation

In your nextQuestion and previousQuestion functions, add:

if (matchesSm) { window.scrollTo({ top: questionRef.current.offsetTop + 75, behavior: 'smooth' }); }

This finds the element’s position and scrolls the window smoothly to it.

➡️ Also on First-Page Select

When selecting the very first question, call the same scroll logic in your handleSelect case.

🎉 Result

Users on a phone will jump directly to each question’s title as they move through—no extra swipes! 🚀

💡 Extra Tip

React refs aren’t just for forms—use them whenever you need to interact with real DOM APIs like scrolling, focus, or measuring! 🛠️
