🌓 What Is Lazy Loading?

• Only load or render what the user sees right now 👀

• Defers off-screen content until needed, so pages start faster 🚀

• Great for long lists, many images, or heavy components 🖼️

⚡ Why Use It?

• Reduces initial bundle size—faster first paint 🏎️

• Saves bandwidth if user never scrolls to some content 💾

• Improves perceived performance & SEO scores 📈

📦 Popular React Package

We’ll use react-lazy-load-image-component—easy & feature-rich.

• ~35k weekly downloads 👍

💻 Install

npm install react-lazy-load-image-component

🖼 Lazy Loading an Image

Instead of:

<img src="photo.jpg" alt="..." />

Use:

import { LazyLoadImage } from 'react-lazy-load-image-component';

<LazyLoadImage
  src="photo.jpg"
  alt="My Photo"
  width="300"
  height="200"
/>

🔧 Key Props

• placeholder: React element while loading ⏳

• placeholderSrc: image URL as placeholder 📷

• threshold: pixels before viewport to start loading 📐

• effect: apply fade/blur/wave on load ✨

📝 Usage Tips

1. Always set width/height to avoid layout shifts 📏

2. Tune threshold so images load just in time ⌛

3. Use a lightweight placeholder or low-res thumbnail 🔍

4. Wrap heavy components in LazyLoadComponent too 📦

🚀 Refactoring Your App

• Identify all long lists and off-screen images 🗒️

• Replace <img> with <LazyLoadImage>

• Wrap non-visible sections/components in <LazyLoadComponent>

• Test responsiveness & scroll behavior ✔️

🎓 Takeaways

• Lazy loading boosts performance, especially on mobile 📱

• react-lazy-load-image-component makes it easy 👍

• Use placeholders & threshold for smooth UX 😊
