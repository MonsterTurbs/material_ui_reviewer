💻 Custom Software Dev Page

Beginner-friendly breakdown with emojis! 📚

🗂️ 1. Page Structure Overview

Our custom software page has three main parts stacked top to bottom (column): 1️⃣ Top nav arrows + title & text 2️⃣ Three feature icons (Save Energy/Time/Money) 3️⃣ Two animations section 4️⃣ Root cause analysis block 5️⃣ Two more animations block

🚀 2. Main Wrapper

  • Use a <Grid container direction="column"> wrapping the entire page.
  • This stacks each section vertically.

⬅️➡️ 3. Top Nav & Text

  • Inside main column container, create a row container for arrows + text.
  • Grid items in this row: • Left arrow (image) • Text block (column container with H2 + paragraphs) • Right arrow (image)
  • Hide arrows on mobile with a Hidden component (MD down).

🔧 4. Feature Icons Row

  • Create a Grid container direction="row" justify="space-around".
  • Each feature (Energy, Time, Money) is its own item container: • A column container: title (H4) above icon (image).

🌳 5. Root Cause Analysis Block

  • One row container centered (justify="center").
  • Single column container: • Tree image • Text block (H4 + paragraphs) stacked below

🎞️ 6. Two Animations Section

  • Row container with justify="space-around".
  • First animation item: • Column container: text (H4 + paragraph) then Lottie animation.
  • Second animation item: • Column container: Lottie animation then text (H4 + paragraph).

💡 7. Mobile Adjustments

  • Hide desktop arrows at MD and below.
  • Switch row→column for small screens: direction={matchesSM ? "column" : "row"}.
  • Center text & buttons on mobile: justify={matchesSM ? "center" : "space-between"}.

🎉 8. Executive Tweaks

  • Moved forward arrow next to paragraph for better UX.
  • Kept arrows hidden on mobile.
  • Made design decisions based on testing & consistency.
