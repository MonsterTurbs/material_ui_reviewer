🔧 Table Issues & Fixes Study Notes 🔧

📌 Problem 1: Too Many Rows Showing

When we added more projects, the table showed more than the allowed rows per page.

Fix: Make sure each row has a unique key (e.g., different names).

✅ Always give rows unique IDs!

📌 Problem 2: Footer Issue

The footer didn't stay at the bottom when there were fewer rows.

Fix: Increased margin at the bottom from 15m to 35m.

✅ More space ensures footer stays correctly at bottom!

📌 Problem 3: Wrong Pagination Count

Pagination didn't update properly with search results.

Fix: Adjusted table pagination to only count rows that match the search filter.

✅ Pagination now correctly reflects filtered data!

📌 Problem 4: Blank Pages Showing

When searching from page 2, results were blank or incorrect.

Fix: Made search always jump back to the first page.

✅ Users always see the correct results immediately!

🎯 Important Tips:
  • Always use unique keys for table rows 🔑
  • Adjust margins to avoid UI problems 📐
  • Ensure pagination matches filtered data 🔍
  • Reset page number when searching 🧹
