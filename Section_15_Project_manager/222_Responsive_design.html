🎯 Goal

Make our data table and page layout look great on all screen sizes 📱💻 by using Material‑UI’s responsive utilities and CSS tweaks.

🏷️ Detect Breakpoints

Use useMediaQuery with theme breakpoints to check screen width:

const matchesMd = useMediaQuery(theme.breakpoints.down('md')); const matchesSm = useMediaQuery(theme.breakpoints.down('sm'));

These booleans tell us when viewport is at or below medium (md) or small (sm) sizes.

↕️ Adjust Spacing at Medium

Change bottom margin under the table for medium screens:

<Grid item mb={matchesMd ? '40m' : '35m'}>…</Grid>

↔️ Enable Horizontal Table Scroll

Prevent overflow by constraining the table container:

.tableWrapper { max-width: 100%; overflow-x: auto; }

🔀 Switches Layout on Small

Wrap switch controls in a Grid and change layout direction:

<Grid container direction={matchesSm ? 'column' : 'row'} justify={matchesSm ? 'center' : undefined} > <Grid item><Switch … /></Grid> </Grid>

⬅️ Label Placement

Move labels left on small screens for neat alignment:

<FormControlLabel control=<Switch … /> labelPlacement={matchesSm ? 'end' : 'start'} />

🔧 Responsive Margins

Zero out left/right margins on small screens:

sx={{ ml: matchesSm ? 0 : '5m', mr: matchesSm ? 0 : '5m' }}

Apply on Grid items and form labels to keep center alignment.

📏 Adjust Search Field Width

Resize search input for small screens:

<TextField sx={{ width: matchesSm ? '25m' : '35m' }} … />

🎉 Result

Your table now:

  • Maintains proper spacing under footer 📐
  • Allows horizontal scroll without layout breakage ↔️
  • Stacks filters neatly on small screens 🗂️
  • Centers content and inputs for mobile 📱
