🎨 Study Notes: Material UI Theme - Palette

🧠 What is the Palette?

The palette in Material UI is where you define your application's color scheme. It helps create a consistent look and feel across your app.

📦 Palette Properties

  • common – stores shared colors (e.g. brand colors)
  • type – set to 'light' or 'dark'
  • primary – your app's main color; auto-generates light and dark shades
  • secondary – used for accents; works just like primary
  • error – colors used when showing error states
  • grey – shades of grey from light to dark (e.g. grey[100], grey[900])
  • text – different opacities for primary, secondary, and disabled text
  • divider – line separators with subtle opacity
  • background – controls the default page and component backgrounds
  • action – defines opacity for hover, selected, disabled states

🎯 Why Use the Palette?

  • Centralizes all colors in one file
  • Easy to change colors across the whole app
  • Helps maintain consistency
  • Material UI uses the palette by default in many components

🧪 Example of a Custom Palette

const theme = createMuiTheme({
  palette: {
    type: 'light',
    primary: {
      main: '#007bff',
    },
    secondary: {
      main: '#ff4081',
    },
    background: {
      default: '#f5f5f5',
      paper: '#ffffff',
    },
    text: {
      primary: '#000',
      secondary: '#555',
    },
  }
});

📝 Summary

  • The palette is part of the theme object
  • It controls all colors used in your app
  • You can override default values by setting them in your custom theme
  • Now that we understand the structure, we’re ready to create our own theme!
