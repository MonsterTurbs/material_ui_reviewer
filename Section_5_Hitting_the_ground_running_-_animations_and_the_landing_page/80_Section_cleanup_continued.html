🛠️ Landing Page Buttons & Cleanup

Simple steps for wiring buttons, setting active tabs, and fixing little bugs!

🔢 1. Add onClick to Hero “Estimate”

In LandingPage.js, for the Hero block’s Estimate button:

onClick={() => props.setValue(5)} (5 clears the active tab)

Remember: wrap it in an arrow function so it only fires on click.

🔄 2. Wire All “Learn More” Links

Copy that same onClick handler to every Learn More button:

  • Revolution → props.setValue(2)
  • Services (Custom Software) → props.setValue(1); props.setSelectedIndex(1)
  • iOS & Android → props.setValue(1); props.setSelectedIndex(2)
  • Website Dev → props.setValue(1); props.setSelectedIndex(3)
  • About Us → props.setValue(3)
  • Contact Us → props.setValue(4)

📨 3. Pass setValue to CallToAction

In LandingPage.js, when rendering your CallToAction component:

<CallToAction setValue={props.setValue} />

Then inside CallToAction.js add props and use onClick={() => props.setValue(2)} for its Learn More, and props.setValue(5) for its Estimate button.

🐞 4. Fix Info-Block Click Glitch

The “About Us” buttons sometimes needed two clicks because we used position: absolute on that grid item. Fix: remove the extra absolute-div and instead apply the background class to the grid container. No more weird jumps! 👍

🚫 5. Remove Horizontal Scroll Bug

If your page ever scrolls sideways, inspect the elements covering >100% width. In our Info block, replace the spacing on the grid container (which adds extra pixels) with a simple margin-bottom on the inner item when stacked:

margin-bottom: {matchesXX ? '10rem' : '0'}
You did it! 🎉 Go celebrate your polished landing page! 🚀
