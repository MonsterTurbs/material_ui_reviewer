📋 Call to Action: Background & Layout Notes

🖼️ 1. Add Background Div

  • Above your main <Grid container>, insert: <div className={classes.background} /> 😊
  • This div will hold the desktop & mobile background images.

📥 2. Import Images

  • Import desktop image: import background from 'assets/background.jpg'
  • Import mobile image: import mobileBackground from 'assets/mobileBackground.jpg' 📱

🎨 3. Define CSS Styles

  • In your makeStyles object, add: 
    background: {
  backgroundImage: `url(${background})`,
  backgroundSize: 'cover',
  backgroundPosition: 'center',
  height: '60rem',
},
  • Under a mobile breakpoint, override: 
    [theme.breakpoints.down('md')]: {
  backgroundImage: `url(${mobileBackground})`
}

🔄 4. Move Background to Container

Instead of using an extra div, you can attach the background class directly to your main <Grid container>. Then remove any position: absolute on grid items to keep layout natural.

⚙️ 5. Layout Buttons & Text

  • Wrap your heading and subtitle in one Grid item.
  • Wrap the “Free Estimate” <Button> in another Grid item.
  • Use justify="space-between" on the container to spread them apart.

🎨 6. Style the Button

  • Extend your theme’s estimate style:
    • estimateButton: {
  ...theme.typography.estimate,
  backgroundColor: theme.palette.common.orange,
  borderRadius: 50,
  height: '8rem',
  width: '20.5rem',
  marginRight: '5rem',
  fontSize: '1.5rem'
}
  • Adjust marginRight so it doesn’t touch the screen edge 😊.

📏 7. Center Text Vertically

Use alignItems="center" on the grid container (row direction) to move text into the vertical middle of the background.

📱 8. Responsive Check!

  • Resize the browser: at md breakpoint, mobile background should appear.
  • Ensure text & button stay readable and well-spaced on small screens.

✅ 9. Final Review

  • Background covers full container on desktop and mobile.
  • Text and button align with other page elements.
  • Everything remains clear, fun, and friendly—even for beginners! 🎉
