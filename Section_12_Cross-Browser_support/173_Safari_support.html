🧐 What Happened in Safari?

Safari won't show text inside an SVG when the SVG is used in an <img> tag. This made our header text disappear! 😲

🖼️ Inline SVG Fix

  • 👉 Copy the SVG code and paste it directly into your React component instead of using <img src="Logo.svg">.
  • 👉 Add className={classes.logo} to the <svg> so styles still apply.

💅 Fixing Styles in React

  • 🤔 React doesn't allow CSS inside a normal <style> tag with braces.
  • 💡 Wrap the CSS inside backticks and curly braces: {`...CSS here...`}.
  • ✅ Remove unsupported quotes in font-family and add missing properties like fontWeight or textTransform.

🔄 React DOM Property Fixes

  • 🚫 Change class="..." to className="...".
  • 🚫 Change font-size="..." to fontSize="..." (camelCase).
  • ✔️ These small updates clear console errors and let React render correctly. ✅

🔤 Heading Consistency

We switched some <h2> tags to <h1> so styles match and Safari shows the right size text. 📝

📱 Fixing Icon Overlap

  • 🐛 In Safari, our Lottie icons were cut off at certain widths.
  • 🔧 Reduce maxWidth at medium breakpoints (e.g., from 20rem to 15rem) and set explicit height.

🔧 Restoring Grid Layout

  • ⚙️ A style change turned our icon container from display: grid into block.
  • 🔄 Add style={{ display: 'grid' }} back on the container to re-align icons. 🎯

🔍 Key Takeaways

  • 🔄 Always test your site in Safari (Mac + iOS) as well as Chrome.
  • 🐛 Console errors often give clues (invalid DOM props).
  • ⚙️ Small changes (inline SVG, props fixes, CSS tweaks) keep your site looking great everywhere!
